# SocialEval MVP 云上部署配置单

## 文档目的

本文档用于给 `SocialEval` 一期 MVP / 最小验证阶段提供一套可直接下单的云资源配置建议，优先面向**上海交通大学交大云**，并给出**腾讯云**与**阿里云**的对应规格，便于后续横向对比。

适用前提：
- 当前阶段以 `v1 / MVP` 为目标
- 不追求高可用集群，只要求稳定跑通核心链路
- 已确认系统使用外部 AI 模型 API，不需要本地 GPU 推理
- 已确认系统需要保留完整原始记录、报告版本和访问审计
- 校内已有对象存储能力可用

---

## 推荐结论

MVP 阶段推荐采用**单机部署 + 对象存储**的最小可行方案：

| 项目 | 推荐值 |
|------|--------|
| 云主机规格 | `8 vCPU / 32 GB RAM` |
| 系统盘 | `100 GB SSD` |
| 数据盘 | `200 GB SSD` |
| 对象存储 | `1 个私有 Bucket` |
| 操作系统 | `Ubuntu 22.04 LTS` |
| 部署模式 | 单机部署 `Nginx + FastAPI + Celery + PostgreSQL + Redis` |

选择理由：
- 本系统当前不需要 GPU，核心压力来自 Web/API、异步任务、文件解析、数据库、缓存与审计写入
- `4核8G` 或 `4核16G` 虽可勉强运行，但在文档解析、后台任务和 PostgreSQL/Redis 同机部署时，稳定性偏弱
- `8核32G` 是兼顾稳定性、复杂度和可维护性的单机起步点

## 交大云配置单

### 推荐规格

| 资源 | 数量 | 规格 | 用途 | 是否必需 |
|------|---:|------|------|------|
| 云主机 | 1 | 增强款 通用型 `8核32GB` | 单机运行 API、前端静态资源、Celery、PostgreSQL、Redis | 必需 |
| 系统盘 | 1 | 性能型 `100GB` | 操作系统、运行环境、容器/服务程序 | 必需 |
| 数据盘 | 1 | 性能型 `200GB` | 数据库、Redis 持久化、日志、临时文件 | 必需 |
| 对象存储 | 1 | 私有 Bucket | 原始文件、AI 原始长文本、报告快照、PDF/JSON 导出 | 必需 |
| 安全组 | 1 | 默认安全组 + 最小放通 | 端口控制 | 必需 |

### 交大云下单建议

- 云主机：选择**增强款**
  - 原因：交大云官方说明增强款“性能稳定且资源独享”，比基础款更适合长期跑后台任务和数据库
- 规格：选择**通用型 8核32GB**
  - 原因：MVP 阶段单机需要同时承载应用、任务、数据库和缓存，通用型更均衡
- 磁盘：
  - 系统盘 `100GB`
  - 数据盘 `200GB`
- 网络：
  - 若主要面向校内访问，优先使用 `校园网`
  - 如需校外直接访问，再由网络策略单独放通或通过统一入口代理

### 单机部署角色划分

这台机器建议同时运行：

- `Nginx`
- `FastAPI`
- `Celery Worker`
- `PostgreSQL`
- `Redis`

对象存储用于保存：

- 论文原文件
- 预处理产物
- AI 调用原始长文本
- 报告 JSON
- 报告 PDF
- 审计归档文件

### 端口与安全建议

- 对外开放：
  - `80`
  - `443`
- 仅管理员 IP 放通：
  - `22`
- 不对公网开放：
  - `5432`
  - `6379`

## 腾讯云对应配置

| 资源 | 数量 | 规格 | 用途 |
|------|---:|------|------|
| 云主机 | 1 | `CVM SA5 8核32GB` | 对应单机 MVP 主机 |
| 系统盘 | 1 | `100GB SSD` | 系统与运行环境 |
| 数据盘 | 1 | `200GB SSD` | 数据与日志 |
| 对象存储 | 1 | `COS 私有 Bucket` | 原始文件与报告快照 |

说明：
- 若在腾讯云上复刻同一 MVP 架构，`SA5 8核32GB` 是与交大云 `8核32GB` 通用型较接近的对应选择
- 部署模式同样建议单机运行 `Nginx + FastAPI + Celery + PostgreSQL + Redis`

## 对象存储目录建议

建议在对象存储内按用途分目录管理：

- `papers/raw/`
- `papers/processed/`
- `ai-calls/raw-input/`
- `ai-calls/raw-output/`
- `reports/json/`
- `reports/pdf/`
- `audit-archive/`

---

## 备份建议

MVP 阶段虽然是单机，但仍建议至少具备以下备份能力：

- PostgreSQL 每日备份
- 数据盘定期快照
- 对象存储开启版本控制或保留恢复策略
- 应用日志与审计日志定期归档

---

## 不推荐的更低配置

以下配置仅适合非常轻的 demo，不推荐作为正式 MVP 验证环境：

- `4核8GB`
- `4核16GB`

原因：
- 单机同跑 PostgreSQL、Redis、Celery 与文档解析时，资源冗余太小
- 当前需求已经确认需要完整原始记录、报告版本和访问审计，磁盘 I/O 与内存压力会高于普通演示系统

---

## 最终建议

如果当前就要下单，建议直接选择：

- **交大云 增强款 通用型 `8核32GB`**
- **系统盘 `100GB`**
- **数据盘 `200GB`**
- **校内对象存储 `1 个私有 Bucket`**

这就是 `SocialEval` 一期 MVP / 最小验证阶段最合适的精简版云资源配置。
