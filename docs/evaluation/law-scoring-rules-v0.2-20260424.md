## 一、基本判断

现有六维框架可以保留，但不宜理解为六个完全并列、完全独立、等强度的指标。更合适的定位是：六维对应法学论文人工评价中的一条判断链。

参考 `ref/法学论文的人工评价逻辑.pptx`，人工评价链条可以概括为：

> 问题新颖性 → 到达点与未竟点 → 类型化能力 → 推理连贯性 → 共同体可接受性 → 共同体开放性

对应到当前六维：

| 判断链条 | 当前维度 | 执行含义 |
|---|---|---|
| 问题新颖性 | 研究创新性 | 是否通过问题创新、内容/观点创新或方法/材料创新形成真实研究贡献 |
| 到达点与未竟点 | 现状洞察度 | 是否讲清既有研究已经解决什么、尚未解决什么 |
| 类型化能力 | 分析框架建构力 | 是否形成可操作的概念、类型、解释框架或制度推导工具 |
| 推理连贯性 | 逻辑严密性 | 论证推进是否连贯、充分、可检验 |
| 共同体可接受性 | 结论可接受性 | 结论是否存在制度跳跃、口号式对策或与前文矛盾等风险信号（AI 初筛只做风险筛查，不裁定最终可接受性） |
| 共同体开放性 | 前瞻延展性 | 是否为后续研究、解释路径或制度讨论打开空间 |

## 二、三层结构

六维应按三层理解，而不是平铺计分。

| 层级 | 维度 | 作用 |
|---|---|---|
| 基础层 | 研究创新性、现状洞察度、分析框架建构力、逻辑严密性 | 判断论文是否"站得住"，以及是否构成真实知识推进 |
| 约束层 | 结论可接受性 | 筛查结论层面的制度风险信号，决定总分上限 |
| 外溢层 | 前瞻延展性 | 判断论文是否在既有论证基础上打开后续研究或制度讨论空间（弱加分） |

基础层塌陷时，约束层和外溢层不能补救总分。前瞻延展性尤其不应奖励脱离正文论证的宏大展望。

## 三、全局评分协议

每一维按 0-100 分评分，但不得让模型直接凭整体印象打分。每维必须先完成以下判断：

| 字段 | 要求 |
|---|---|
| 核心判断 | 说明这一维到底在评什么，不混入其他维度 |
| 证据片段 | 必须来自论文原文、注释、参考文献或论证结构 |
| 检查清单结果 | 逐项说明是否满足（与 AI 辅助评审规程的检查清单一致） |
| 缺陷触发 | 判断是否触发跨维度上限规则 |
| 边界说明 | 仅限研究创新性、现状洞察度、结论可接受性、前瞻延展性，明确本次判断的是哪一层，不判断哪一层 |
| 最终分数 | 在分档和上限约束后给出 |

两项特殊要求：

- **结论可接受性**：优先输出"低风险/中风险/高风险"以及对应的总分上限结果，不建议只写"可接受/不可接受"
- **前瞻延展性**：优先输出"是否加分 + 延展路径摘要"，不建议把弱信号维度细分到过高精度

## 四、权重与评分公式

### 4.1 基础分权重

- 研究创新性：`30`
- 现状洞察度：`20`
- 分析框架建构力：`15`
- 逻辑严密性：`20`

合计：`85`

### 4.2 修正规则

- 结论可接受性：作为总分上限项
- 前瞻延展性：作为弱加分项，最高 `5` 分

### 4.3 评分公式

```text
基础分 = (研究创新性×30 + 现状洞察度×20 + 分析框架建构力×15 + 逻辑严密性×20) / 85

前瞻加分条件：逻辑严密性 ≥ 60、结论可接受性 ≥ 60、主评分维度无塌陷项（< 50）
前瞻加分标准：0 / 2-3 / 4-5（原则上不使用 1 分）

结论可接受性上限：
  ≥ 75：不上限
  60-74：总分上限 75
  < 60：总分上限 65

最终分 = min(基础分 + 前瞻加分, 结论可接受性上限)
```

## 五、硬约束规则

为防止 AI 被"包装型论文"或"维度错位"误导，适用以下硬规则（按维度分组）：

### 5.1 全局规则

- 任一主评分维度 `< 50` 时，前瞻延展性不得加分
- 不得使用前瞻延展性补救前四维缺陷
- 不得使用结论可接受性补救逻辑塌陷
- 存在"材料与结论严重错位"时，必须标记为需人工复核
- 不得在无文内证据的情况下给出高分；证据不足时必须说明"不足"，不得用想象补齐
- 各维度检查清单应避免重复计分，同一判断点只在最相关的维度中评价一次

### 5.2 研究创新性

- 不得因"题目宏大""现实重大""首次提出""填补空白"等自述直接判为高分
- 不得把"创新主张表述清楚"本身等同于高创新性；必须看是否形成真实研究推进
- 不得将材料或方法的新奇性本身视为创新成立；只有在改变论证质量或结论推进时才可加分
- 方法或材料是否"提升了论证"，归入逻辑严密性判断，不在本维重复计分
- 研究创新性 `< 50` 时，最终分不得高于 `65`

### 5.3 现状洞察度

- 不得以参考文献数量替代现状洞察度
- 不得把"至少几个流派"或"至少几个方向"当作高分充分条件
- 现状洞察度 `< 60` 时，研究创新性不得进入最高档（80-100）

### 5.4 分析框架建构力

- 不得将"引用他人框架"直接判为低分；只要本文明确采用边界并转化为分析工具，仍可高分
- 不得将"方法名词堆砌"判为框架建构
- 不得因未采用显式类型化就直接判为低分；概念界分、规范判断标准、比较法矩阵、教义学解释步骤均可构成框架

### 5.5 逻辑严密性

- 逻辑严密性 `< 50` 时，最终分不得高于 `70`
- 不得把连接词丰富、语句流畅或篇幅完整当作逻辑严密性证据
- 不得因没有单独"反驳"小节直接判为低分，应看限制条件或异议是否被实质处理

### 5.6 结论可接受性

- 不得以"接近主流立场"直接判为高可接受性
- 不得把激进、批判性或非主流结论直接判为低可接受性
- 理论性、批判性论文不因缺少即时实施步骤而自动低分；只有在完全缺乏论证桥梁时才降分
- 不得使用结论可接受性补救逻辑塌陷
- 研究创新性 `≥ 80` 且逻辑严密性 `≥ 75`，但本维 `< 60` 时，不得直接视为差稿，必须进入人工复核

### 5.7 前瞻延展性

- 数量只能作为辅助信号，不能因"列出多个方向"自动高分
- 前瞻延展性显著高于前四维表现时，必须标记为需人工复核

## 六、六维细则

| 维度 | 评什么 | 高分锚点 | 低分锚点 | 不得高分情形 |
|---|---|---|---|---|
| 研究创新性 | 是否形成真实研究贡献（问题、内容/观点、方法/材料至少一项明确推进） | 在问题、内容/观点、方法/材料中至少一项形成明显推进，且通常有两项相互支撑 | 热点搬运、旧题换名、政策命题复述、材料堆砌或观点复述 | 仅凭"首次""填补空白""现实意义重大"自称创新 |
| 现状洞察度 | 是否建立"到达点—未竟点—本文切入点"的研究地图 | 能清楚归纳既有研究已解决什么、未解决什么，并识别至少 `2` 组可区分的立场或争点结构 | 文献罗列、综述流水账、只按时间或作者堆材料 | 以参考文献数量、引用名家数量替代洞察 |
| 分析框架建构力 | 是否形成可被后文调用的分析工具（概念界定、类型化、判断标准、分析步骤） | 形成可操作的概念、类型、比较维度或解释框架，并在后文被实际调用 | 概念混乱、理论拼贴、术语堆砌、只在开头提出框架 | 只贴"治理""协同""比例原则"等标签但不展开规则；方法名词堆砌 |
| 逻辑严密性 | 推理链条是否成立（前后衔接、推理充分、概念一致、材料匹配、限制条件处理） | 每一步论证都有明确依据，后文是对前文的推进而非跳跃，材料支撑结论，限制条件被处理 | 跳步推理、偷换概念、材料和结论错位 | 语言流畅但没有可复述的论证链 |
| 结论可接受性 | 结论是否存在风险信号（与前文矛盾、制度跳跃无桥梁、口号式对策） | 低风险：结论与前文一致，即使涉及制度变革也有论证桥梁；中风险：路径说明不充分；高风险：与前文矛盾或停留在口号层面 | 与前文推理矛盾、明显制度跳跃无桥梁、万能药式对策 | 把"符合主流"直接等同于低风险；把激进结论直接等同于高风险 |
| 前瞻延展性 | 是否基于正文打开后续研究或制度讨论路径 | 至少提出 `1` 条具体延展路径，与正文紧密相关，且说明何以被打开 | 空泛拔高、结尾喊口号、与正文无关的未来展望 | 前四维站不住时，不能靠"前景重大"拿加分 |

## 七、分档锚点

### 研究创新性

本维度统一覆盖三类创新来源：

1. **问题创新**：是否提出了可争辩、可推进的法学问题。
2. **内容/观点创新**：是否提出新的命题、解释结论或论证结果，而不是复述既有立场。
3. **方法/材料创新**：是否明确使用新案例、新史料、新数据、新比较对象或新的材料组织方式。

注意：方法或材料是否"提升了论证质量"属于逻辑严密性维度的判断范围，本维只判断方法或材料是否明确且具有差异性。

| 分数段 | 示例 |
|---|---|
| 90-100 | 在问题、内容/观点、方法/材料中至少两项形成强推进，例如既重组核心问题，又提出新的解释结论或材料证成路径 |
| 75-89 | 至少有一项创新来源明确成立，且能在既有争论内形成实质推进 |
| 60-74 | 论文存在一定创新贡献，但主要体现在材料、场景或表述层面，推进幅度有限 |
| 40-59 | 有研究主题，但创新来源单薄、模糊或主要停留在宣称层面 |
| 0-39 | 没有形成真实研究贡献，只是口号、常识复述或热点拼接 |

### 现状洞察度

| 分数段 | 示例 |
|---|---|
| 90-100 | 能清楚说明既有研究已经解决什么、没有解决什么，并让本文问题变得不可回避；至少识别出 `2` 组可区分的立场或争点结构 |
| 75-89 | 文献掌握扎实，能区分主要观点和争点；至少有 `1` 组明确对象 |
| 60-74 | 有基本综述，但批判性和争点提炼有限 |
| 40-59 | 文献多但只是罗列，未形成研究地图 |
| 0-39 | 基本不掌握相关研究，或虚构研究空白 |

### 分析框架建构力

| 分数段 | 示例 |
|---|---|
| 90-100 | 形成清晰的概念界定、类型化、比较维度或解释框架，并在后文被反复调用，具有可操作性 |
| 75-89 | 概念和框架清楚，能提升分析操作性，但边界或可操作性略有不足 |
| 60-74 | 有框架雏形，但类型之间边界或适用条件不够清楚 |
| 40-59 | 理论术语多，但概念关系松散，只有术语堆砌或方法名词罗列 |
| 0-39 | 理论贴标签，核心概念混乱或互相矛盾，没有可操作的分析工具 |

### 逻辑严密性

| 分数段 | 示例 |
|---|---|
| 90-100 | 每一步论证都有明确依据，推理充分、概念一致、材料匹配、限制条件处理充分 |
| 75-89 | 主线连贯，主要结论有材料和规范支撑，限制条件有所处理 |
| 60-74 | 基本能成立，但存在局部跳步或限制条件处理不足 |
| 40-59 | 论证经常从材料直接跳到结论，推理依据不足 |
| 0-39 | 论证链断裂，关键结论无法由前文推出，存在概念偷换或材料错位 |

### 结论可接受性

本维不采用常规分档，而是按风险等级划分：

| 风险等级 | 分数范围 | 总分上限 | 示例 |
|---|---|---|---|
| 低风险 | 75-100 | 不上限 | 结论与前文推理一致，即使涉及制度变革也有论证桥梁和适用边界 |
| 中风险 | 60-74 | 75 | 存在一定风险信号，或制度变革主张的路径说明不充分，需人工评估 |
| 高风险 | < 60 | 65 | 结论与前文明显矛盾，或明显制度跳跃无桥梁，或基本停留在口号层面 |

### 前瞻延展性

本维不采用常规分档，而是按加分等级划分：

| 加分等级 | 分值 | 示例 |
|---|---|---|
| 高加分 | 4-5 | 至少提出 `1` 条具体且与正文紧密相关的延展路径，能明确指出下一轮研究或争论从何处展开 |
| 低加分 | 2-3 | 提出了延展方向，但路径模糊、相关性不足 |
| 不加分 | 0 | 空泛展望（如"有待进一步研究"）或口号式拔高 |

原则上不使用 `1` 分，避免在低信号维度制造伪精度。

加分前置条件：逻辑严密性 `≥ 60`、结论可接受性 `≥ 60`、主评分维度无塌陷项（< 50）。

## 八、典型好坏论文区分

### 好论文

好论文不是六个标签都写到了，而是形成了递进链条：

1. 抓住真问题，明确说明研究创新贡献。
2. 说明既有研究的到达点与未竟点，建立研究地图。
3. 形成可操作的分析工具（概念、类型、比较维度或解释框架），并在后文调用。
4. 让每一步论证都有明确依据，后文是对前文的推进而非跳跃。
5. 结论与前文推理一致，即使提出制度变革也有论证桥梁。
6. 为下一轮研究或制度讨论打开具体路径。

### 坏论文

坏论文通常不是完全没有内容，而是在链条中出现断裂：

1. 起点溃散：有热点主题但没有真问题，只有创新宣称无具体推进。
2. 文献失焦：有引用但没有到达点与未竟点，只有罗列无研究地图。
3. 工具失效：有理论术语但只是贴标签或方法名词堆砌，没有可操作分析工具。
4. 骨架错位：段落顺序存在，但论证关系不成立，推理跳跃或概念偷换。
5. 结论空泛：制度建议变成"完善立法、加强监管、健全机制"，或与前文推理矛盾。
6. 包装拙劣：展望口号化，与正文论证脱节，前四维站不住却试图靠展望加分。

## 九、后续落地建议

1. 短期保留六维名称，不新增第七维。
2. 在 `configs/frameworks/law_v1.yaml` 中把各维 `prompt_template` 从单句评估要点改写为包含"评什么、不评什么、上限规则、分档锚点、证据要求、检查清单"的评分规程。
3. 输出结构可在现有字段基础上补充 `strengths`、`weaknesses`、`limit_rule_triggered`、`confidence_signals`；如不改 schema，则先写入 `analysis` 字段。
4. 用正负样本做校准：PPT 中三篇正样本应呈现不同维度强项；负样本应在对应断裂节点被明显压分。
5. 监测六维分数相关性：如果不同论文中六维分数长期高度趋同，说明框架区分度不足，需要继续调整维度边界或权重。
