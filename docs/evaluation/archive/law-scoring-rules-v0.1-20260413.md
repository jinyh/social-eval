# 法学六维评价评分细则 v0.1

**日期**：2026-04-13  
**定位**：面向 SocialEval 法学论文评价框架的评分规程讨论稿  
**目标**：提升“好论文”和“坏论文”的区分度，避免模型被热点题目、文献堆砌、理论术语、流畅表达或口号式展望误导。

**对齐说明（2026-04-24）**：为与 `law-ai-assisted-review-rules-v0.1-20260424.md` 保持一致，本文将第一维统一表述为“研究创新性”。其中，“问题创新”仍是第一维的起点与核心子项，但不再被理解为创新的唯一来源。

## 一、基本判断

现有六维框架可以保留，但不宜理解为六个完全并列、完全独立、等强度的指标。更合适的定位是：六维对应法学论文人工评价中的一条判断链。

参考 `ref/法学论文的人工评价逻辑.pptx`，人工评价链条可以概括为：

> 问题新颖性 → 到达点与未竟点 → 类型化能力 → 推理连贯性 → 法学共同体可接受性 → 法学共同体开放性

对应到当前六维：

| 判断链条 | 当前维度 | 执行含义 |
|---|---|---|
| 问题新颖性 | 研究创新性 | 是否通过问题创新、内容/观点创新或方法/材料创新形成真实研究贡献 |
| 到达点与未竟点 | 现状洞察度 | 是否讲清既有研究已经解决什么、尚未解决什么 |
| 类型化能力 | 理论建构力 | 是否形成概念、类型、规范解释或制度推导框架 |
| 推理连贯性 | 逻辑严密性 | 论证推进是否连贯、必要、可检验 |
| 共同体可接受性 | 学术共识度 | 是否进入法学共同体争论，并具备可被认真回应的理由 |
| 共同体开放性 | 前瞻延展性 | 是否为后续研究、解释路径或制度讨论打开空间 |

## 二、三层结构

六维应按三层理解，而不是平铺计分。

| 层级 | 维度 | 作用 |
|---|---|---|
| 基础层 | 研究创新性、现状洞察度、理论建构力、逻辑严密性 | 判断论文是否“站得住”，以及是否构成真实知识推进 |
| 共同体层 | 学术共识度 | 判断论文是否进入法学共同体可理解、可争辩、可回应的语境 |
| 外溢层 | 前瞻延展性 | 判断论文是否在既有论证基础上打开后续研究或制度讨论空间 |

基础层塌陷时，后两层不能补救总分。前瞻延展性尤其不应奖励脱离正文论证的宏大展望。

## 三、全局评分协议

每一维按 0-100 分评分，但不得让模型直接凭整体印象打分。每维必须先完成四类判断：

| 字段 | 要求 |
|---|---|
| 核心判断 | 说明这一维到底在评什么，不混入其他维度 |
| 证据片段 | 必须来自论文原文、注释、参考文献或论证结构 |
| 缺陷触发 | 判断是否触发跨维度上限规则 |
| 最终分数 | 在分档和上限约束后给出 |

分档建议：

| 分档 | 含义 |
|---|---|
| 90-100 | 卓越：不仅做到该维要求，而且显著推动后续论证或共同体讨论 |
| 75-89 | 强：该维清楚、扎实、有说服力，但未达到重置问题或方法的程度 |
| 60-74 | 合格：能看出该维努力，但深度、清晰度或证据支撑有限 |
| 40-59 | 弱：形式上有该维内容，但实质贡献不足或明显不稳定 |
| 0-39 | 失效：基本缺席、概念混乱、逻辑矛盾，或主要靠口号支撑 |

## 四、跨维度上限规则

这些规则用于识别包装型坏论文。一旦触发，上限优先于正常印象分。

| 触发缺陷 | 典型表现 | 上限规则 |
|---|---|---|
| 没有真问题 | 只有热点主题、政策背景、宏大口号，无法转化为可争辩的法学问题 | 研究创新性 <= 50；前瞻延展性 <= 50；总分通常 <= 65 |
| 研究空白未证明 | 只说“已有研究不足”，但没有说明既有研究到达点和未竟点 | 现状洞察度 <= 60；研究创新性 <= 70 |
| 文献堆砌 | 引用多，但不区分观点流派、争点、代表文献和未解问题 | 现状洞察度 <= 55；学术共识度 <= 60 |
| 理论贴标签 | 套用“大词”，但没有概念界定、类型化或规范推导 | 理论建构力 <= 55；前瞻延展性 <= 65 |
| 论证链断裂 | 后一步不是前一步的必要回应，材料不能支撑结论 | 逻辑严密性 <= 50；学术共识度 <= 60；总分通常 <= 70 |
| 万能药式对策 | 结论停留在“完善立法、加强监管、健全机制” | 学术共识度 <= 65；前瞻延展性 <= 50 |
| 自说自话 | 不回应法学共同体已有争论、判例、规范体系或关键反对意见 | 学术共识度 <= 55；前瞻延展性 <= 60 |
| 口号式展望 | 展望与正文论证脱节，只做宏大拔高 | 前瞻延展性 <= 45 |

说明：上限规则是缺陷触发式规则，不是简单线性依赖。某些论文研究创新性不高，但逻辑论证仍可较强；某些论文理论建构不宏大，但教义学推理仍可扎实。只有出现上述致命缺陷时，才触发相应上限。

## 五、六维细则

| 维度 | 评什么 | 高分锚点 | 低分锚点 | 不得高分情形 |
|---|---|---|---|---|
| 研究创新性 | 是否形成真实研究贡献 | 在问题、内容/观点、方法/材料三方面至少有一项形成明显推进，且通常有两项相互支撑 | 热点搬运、旧题换名、政策命题复述、材料堆砌或观点复述 | 仅凭“首次”“填补空白”“现实意义重大”自称创新 |
| 现状洞察度 | 是否讲清既有研究的到达点与未竟点 | 能准确归纳代表观点、争点结构、既有解释路径及其不足 | 文献罗列、综述流水账、只按时间或作者堆材料 | 以参考文献数量、引用名家数量替代洞察 |
| 理论建构力 | 是否形成可操作的概念、类型、解释框架或制度推导 | 能把复杂法现象类型化，提出可比较模型，或完成从规范到制度方案的推导 | 概念混乱、理论拼贴、术语堆砌 | 只贴“治理”“协同”“比例原则”等标签但不展开规则 |
| 逻辑严密性 | 论证推进是否连贯、必要、可检验 | 问题识别、文献定位、类型化、制度结论之间顺序不可随意颠倒 | 跳步推理、偷换概念、材料和结论错位 | 语言流畅但没有可复述的论证链 |
| 学术共识度 | 是否进入法学共同体的核心争论并具备可接受性 | 准确描述主流与反对立场，即使挑战主流，也给出共同体可认真回应的理由 | 自说自话、选择性忽略关键反驳、只面向政策口号 | 把“符合主流”直接等同于高分 |
| 前瞻延展性 | 是否为后续研究、解释路径或制度讨论打开新空间 | 文章不终结争论，而是重置下一轮争论的问题、路径或反驳位置 | 空泛拔高、结尾喊口号、与正文无关的未来展望 | 前四维站不住时，不能靠“前景重大”拿高分 |

## 六、分档锚点

### 研究创新性

本维度统一覆盖三类创新来源：

1. **问题创新**：是否提出了可争辩、可推进的法学问题。
2. **内容/观点创新**：是否提出新的命题、解释结论或论证结果，而不是复述既有立场。
3. **方法/材料创新**：是否通过新案例、新史料、新数据、新样本或新的材料组织方式，显著提升了论证。

| 分数段 | 示例 |
|---|---|
| 90-100 | 在问题、内容/观点、方法/材料中至少两项形成强推进，例如既重组核心问题，又提出新的解释结论或材料证成路径 |
| 75-89 | 至少有一项创新来源明确成立，且能在既有争论内形成实质推进 |
| 60-74 | 论文存在一定创新贡献，但主要体现在材料、场景或表述层面，推进幅度有限 |
| 40-59 | 有研究主题，但创新来源单薄、模糊或主要停留在宣称层面 |
| 0-39 | 没有形成真实研究贡献，只是口号、常识复述或热点拼接 |

### 现状洞察度

| 分数段 | 示例 |
|---|---|
| 90-100 | 能清楚说明既有研究已经解决什么、没有解决什么，并让本文问题变得不可回避 |
| 75-89 | 文献掌握扎实，能区分主要观点和争点 |
| 60-74 | 有基本综述，但批判性和争点提炼有限 |
| 40-59 | 文献多但只是罗列，未形成研究地图 |
| 0-39 | 基本不掌握相关研究，或虚构研究空白 |

### 理论建构力

| 分数段 | 示例 |
|---|---|
| 90-100 | 提出清晰类型、模型或规范解释框架，并能支撑后文制度推导 |
| 75-89 | 概念和框架清楚，能提升分析操作性 |
| 60-74 | 有框架雏形，但类型之间边界或适用条件不够清楚 |
| 40-59 | 理论术语多，但概念关系松散 |
| 0-39 | 理论贴标签，核心概念混乱或互相矛盾 |

### 逻辑严密性

| 分数段 | 示例 |
|---|---|
| 90-100 | 每一步论证都是对前一步问题的必要回应，反例和限制条件处理充分 |
| 75-89 | 主线连贯，主要结论有材料和规范支撑 |
| 60-74 | 基本能成立，但存在局部跳步或反驳不足 |
| 40-59 | 论证经常从材料直接跳到结论 |
| 0-39 | 论证链断裂，关键结论无法由前文推出 |

### 学术共识度

| 分数段 | 示例 |
|---|---|
| 90-100 | 准确进入共同体核心争论，并能改变争论方式或解释基准 |
| 75-89 | 能回应主流观点、反对观点和实务约束 |
| 60-74 | 有对话意识，但回应对象和争点不够充分 |
| 40-59 | 只选择方便回应的观点，忽略关键反驳 |
| 0-39 | 基本自说自话，无法被共同体定位和回应 |

### 前瞻延展性

| 分数段 | 示例 |
|---|---|
| 90-100 | 为后续研究或制度解释画出清楚地图，反对者也知道应从哪里反驳 |
| 75-89 | 能提出有根据的后续问题或制度路径 |
| 60-74 | 有一定延展价值，但仍较依赖一般性判断 |
| 40-59 | 展望比较空泛，与正文支撑关系弱 |
| 0-39 | 只是口号式展望或万能药式建议 |

## 七、典型好坏论文区分

### 好论文

好论文不是六个标签都写到了，而是形成了递进链条：

1. 抓住真问题。
2. 说明既有研究的到达点与未竟点。
3. 通过类型化、概念框架、规范解释或制度推导提升分析操作性。
4. 让后一步论证成为前一步问题的必要回应。
5. 进入法学共同体已有争论，并能被认真回应。
6. 不终结争论，而是为下一轮研究或制度讨论画出地图。

### 坏论文

坏论文通常不是完全没有内容，而是在链条中出现断裂：

1. 起点溃散：有热点主题但没有真问题。
2. 文献失焦：有引用但没有到达点与未竟点。
3. 工具失效：有理论术语但只是贴标签。
4. 骨架错位：段落顺序存在，但论证关系不成立。
5. 落脚空泛：制度建议变成“完善立法、加强监管、健全机制”。
6. 包装拙劣：表达冗长、对话缺失、结尾口号化。

## 八、后续落地建议

1. 短期保留六维名称，不新增第七维。
2. 在 `configs/frameworks/law_v1.yaml` 中把各维 `prompt_template` 从单句评估要点改写为包含“评什么、不评什么、上限规则、分档锚点、证据要求”的评分规程。
3. 输出结构可在现有字段基础上补充 `strengths`、`weaknesses`、`limit_rule_triggered`、`confidence_signals`；如不改 schema，则先写入 `analysis` 字段。
4. 用正负样本做校准：PPT 中三篇正样本应呈现不同维度强项；负样本应在对应断裂节点被明显压分。
5. 监测六维分数相关性：如果不同论文中六维分数长期高度趋同，说明框架区分度不足，需要继续调整维度边界或权重。
