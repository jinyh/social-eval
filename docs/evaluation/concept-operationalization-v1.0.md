# 法学评价框架核心概念操作化定义 v1.0

## 文档说明

**目的**：将 v2.6 框架中的主观概念转化为可操作的检查清单，提高 AI 评分的稳定性和可重复性。

**原则**：
1. **可验证**：每个标准都能通过文本搜索或段落定位验证
2. **可量化**：尽可能使用数量指标（如"至少 2 个"）
3. **可示例**：提供正面和反面示例
4. **可区分**：明确与相邻概念的边界

**版本**：1.0  
**创建日期**：2026-04-23  
**适用框架**：law-v2.6-20260423.yaml

## 1. 问题创新性维度

### 1.1 核心概念：枢纽性（Pivotal Significance）

#### 原定义（模糊）
> "触及制度、规范或解释上的枢纽位置"

#### 操作化定义（清晰）

**枢纽性 = 满足以下至少 2 项标准**：

□ **标准1：影响范围广**
- 检查方法：问题的解决是否影响多个相关制度/规范
- 搜索关键词："涉及"、"关联"、"影响"、"波及"
- 正面示例："民法典人格权独立成编涉及宪法基本权利体系、民法体系结构、司法适用规则三个层面"
- 反面示例："某地方性法规的具体条款解释"

□ **标准2：争议焦点**
- 检查方法：是否是学界长期争论的核心问题
- 搜索关键词："争议"、"分歧"、"争论"、"对立"
- 正面示例："法律行为与意思表示的关系是民法学界长期争论的核心问题"
- 反面示例："某个新兴概念的首次提出（尚无争论）"

□ **标准3：理论基础**
- 检查方法：是否涉及法学基本原理或核心概念
- 搜索关键词："原则"、"基本"、"核心"、"根本"
- 正面示例："涉及法律行为效力的判断标准，触及意思自治原则"
- 反面示例："某个具体案件的事实认定"

□ **标准4：制度连锁**
- 检查方法：问题的解决是否需要调整多个制度
- 搜索关键词："配套"、"衔接"、"协调"、"体系"
- 正面示例："个人信息保护需要民法、行政法、刑法的协调"
- 反面示例："某个单一制度的技术性调整"

**评分指导**：
- 满足 3-4 项 → 80-100 分（卓越）
- 满足 2 项 → 60-79 分（良好）
- 满足 1 项 → 40-59 分（及格）
- 满足 0 项 → 0-39 分（不及格）

### 1.2 核心概念：可争辩的法学问题

#### 原定义（模糊）
> "可争辩的法学问题，而非政策口号或热点主题"

#### 操作化定义（清晰）

**可争辩的法学问题 = 满足以下所有标准**：

□ **标准1：有明确的问题句**
- 检查方法：能否找到"XX是否YY"、"如何ZZ"、"为什么WW"的问题句
- 搜索关键词："是否"、"如何"、"为什么"、"能否"
- 正面示例："民法典人格权独立成编是否符合宪法基本权利体系？"
- 反面示例："论新质生产力的法治化"（无问题句）

□ **标准2：涉及法律规范**
- 检查方法：是否涉及具体法条、司法解释或判例
- 搜索关键词：《XX法》、"第X条"、"司法解释"、"判例"
- 正面示例："《民法典》第1032条关于隐私权的规定是否涵盖个人信息？"
- 反面示例："应加强网络治理"（无具体法律规范）

□ **标准3：有理论分歧**
- 检查方法：是否存在不同的理论观点或解释路径
- 搜索关键词："有学者认为"、"另一种观点"、"争议"
- 正面示例："对于XX问题，甲说认为...，乙说认为..."
- 反面示例："学界一致认为..."（无分歧）

□ **标准4：非政策倡导**
- 检查方法：是否只是"应该XX"的政策建议，而无法学论证
- 反面关键词："应加强"、"应完善"、"应重视"（无论证）
- 正面示例："基于XX原理，应当采用YY解释路径"（有论证）
- 反面示例："应加强网络安全立法"（无论证）

**触发上限规则**：
- 不满足标准1或标准2 → 最高 50 分
- 不满足标准3或标准4 → 最高 65 分

## 2. 现状洞察度维度

### 2.1 核心概念：研究地图（Research Map）

#### 原定义（模糊）
> "建立研究地图，而不只是罗列观点"

#### 操作化定义（清晰）

**研究地图 = 必须包含以下 3 项要素**：

□ **要素1：到达点（已解决什么）**
- 检查方法：是否明确说明既有研究的贡献
- 要求：至少 2 个具体研究成果 + 其贡献
- 搜索关键词："XX研究指出"、"XX学者认为"、"已有研究表明"、"学界已经"
- 正面示例："张三（2020）通过实证研究证明了XX，李四（2021）进一步揭示了YY"
- 反面示例："已有研究较多"（无具体成果）

□ **要素2：未竟点（缺口在哪）**
- 检查方法：是否明确指出既有研究的不足或未解决的问题
- 要求：明确指出"但尚未解决XX问题"或"仍存在XX争议"
- 搜索关键词："但"、"然而"、"尚未"、"仍然"、"争议"、"不足"、"缺乏"
- 正面示例："但对于XX问题，学界尚未形成共识，主要争议在于..."
- 反面示例："已有研究不足"（未说明具体不足）

□ **要素3：本文切入点（要解决什么）**
- 检查方法：本文的研究问题是否与未竟点直接对应
- 要求：明确说明"本文拟从XX角度解决上述问题"
- 搜索关键词："本文"、"笔者"、"拟"、"试图"、"旨在"
- 正面示例："针对上述争议，本文拟从XX角度，通过YY方法，解决ZZ问题"
- 反面示例："本文研究XX"（未说明与未竟点的关系）

**触发上限规则**：
- 3 项要素均缺失 → 最高 55 分
- 缺失 1-2 项要素 → 最高 65 分

### 2.2 核心概念：学术流派覆盖

#### 原定义（模糊）
> "至少识别出 2-3 个主要流派或代表观点"

#### 操作化定义（清晰）

**学术流派 = 满足以下标准的观点群**：

□ **识别标准**：
- 有明确的代表学者或代表文献
- 有清晰的核心主张
- 与其他流派有明显区别

□ **检查方法**：
1. 搜索关键词："XX学派"、"XX观点"、"有学者认为"、"另一种观点"
2. 统计不同观点的数量
3. 检查是否有对比或争论

□ **评分标准**：
- 识别 3 个及以上流派 → 80-100 分
- 识别 2 个流派 → 60-79 分
- 识别 1 个流派 → 40-59 分
- 无流派识别（只罗列文献）→ 0-39 分

□ **正面示例**：
```
关于XX问题，学界存在三种主要观点：
1. 甲说（代表学者：张三）认为...
2. 乙说（代表学者：李四）认为...
3. 丙说（代表学者：王五）认为...
```

□ **反面示例**：
```
张三（2020）、李四（2021）、王五（2022）等学者对XX问题进行了研究。
（只罗列文献，未区分观点）
```

## 3. 分析框架建构力维度

### 3.1 轨道A：框架可操作性

#### 原定义（模糊）
> "形成可操作的分析工具"

#### 操作化定义（清晰）

**框架可操作性 = 满足以下标准的数量（0-4分）**：

□ **标准1：概念界定是否清晰**
- 检查方法：核心术语是否有明确定义（如"本文所称XX是指…"），内涵和外延可识别
- 搜索关键词："是指"、"定义为"、"包括"、"指的是"、"本文所称"
- 正面示例："本文所称'比例原则'，是指国家干预公民权利时必须满足适当性、必要性、狭义比例性三要件"
- 反面示例："运用同理心正义理论"（未定义什么是同理心正义、其核心要素是什么）

□ **标准2：分类标准是否明确**
- 检查方法：类型划分是否有边界和区分依据（如"根据XX标准可分为…"），不同类型间有明确区分
- 搜索关键词："分为"、"类型"、"可分为"、"根据XX标准"
- 正面示例："根据干预程度，可将行政行为分为轻度干预、中度干预和重度干预三类"
- 反面示例："有多种类型"（无分类标准和边界）

□ **标准3：分析步骤是否可循**
- 检查方法：是否有步骤化的分析流程或判断标准（如"第一步…第二步…第三步…"）
- 搜索关键词："首先"、"其次"、"步骤"、"第一步"、"判断标准"
- 正面示例："判断比例原则需三步：第一步检验适当性，第二步检验必要性，第三步检验狭义比例性"
- 反面示例："需要综合考虑多方面因素"（无具体步骤）

□ **标准4：后文是否实际调用**
- 检查方法：框架核心术语在后文中至少出现3次并发挥分析作用
- 统计方法：核心术语在后文中至少出现 3 次
- 正面示例：绪论提出"四步运作机理"，后文每个章节都使用该框架进行具体分析
- 反面示例：绪论提出框架术语，后文未再提及或仅出现1次

### 3.2 轨道B：法学转化度

#### 核心声明

**跨学科概念只要转化为可操作的法学分析步骤（如裁判规则、司法程序步骤、法律适用流程），就不属于"理论贴标签"。**

法学论文允许引入跨学科概念（心理学、经济学、社会学等），但必须将其转化为法学操作层面可使用的分析步骤。纯粹的跨学科理论移植（只说"运用XX理论"但不转化为法学操作步骤）才是"理论贴标签"。

#### 操作化定义（清晰）

**法学转化度 = 满足以下标准的数量（0-4分）**：

□ **标准1：跨学科概念是否转化为法学分析步骤**
- 检查方法：跨学科概念是否已转化为裁判推理步骤、法律适用流程或法律判断标准
- 搜索关键词："转化为"、"适用于"、"判断流程"、"裁判步骤"、"运作机理"
- 正面示例："同理心正义转化为司法裁判的四步运作机理：移情联想→位阶排序→反思权衡→理由优化"
- 反面示例："运用同理心正义理论分析司法公正"（未说明如何转化为法学操作）

□ **标准2：是否提供裁判规则或司法程序层面的操作指引**
- 检查方法：框架是否包含裁判规则、判断标准或司法适用流程（而非仅停留在理论阐释层面）
- 搜索关键词："法官应"、"裁判规则"、"审查步骤"、"适用标准"、"位阶排序"
- 正面示例："在位阶排序步骤中，法官应按权利强度从高到低排列冲突权利"
- 反面示例："同理心正义要求法官体谅当事人"（停留在情感层面，无裁判操作指引）

□ **标准3：是否在制度设计层面给出可操作方案**
- 检查方法：框架是否指向可操作的制度设计或立法建议（而非仅提出理论主张）
- 搜索关键词："建议"、"制度设计"、"立法建议"、"审查程序"、"规则设置"
- 正面示例："基于反思权衡步骤，建议在立法中设置权利冲突的三级审查程序"
- 反面示例："应完善相关立法"（无具体制度方案）

□ **标准4：转化后的法学操作步骤是否在后文被实际使用**
- 检查方法：法学转化步骤在后文的分析中被反复调用（而非只存在于理论阐述部分）
- 搜索关键词：检查后文是否按转化步骤逐一展开分析
- 正面示例：四步运作机理在后文每个案例分析中都被逐步应用
- 反面示例：理论阐述部分提到四步机理，但案例分析部分并未按步骤展开

### 3.3 评分锚定规则

| 框架可操作性 | 法学转化度 | 分档 |
|-------------|-----------|------|
| >=3 | >=2 | excellent (80-100) |
| >=3 | 1 | good (70-79) |
| >=2 | >=2 | good (60-69) |
| >=2 | <=1 | marginal (50-59) |
| <=1 | 任意 | unacceptable (0-49, 触发上限) |

### 3.4 触发上限规则（修正版）

**no_operational_framework（score_ceiling=50）**：
- 触发条件：核心术语完全无定义 + 无任何可循的分析步骤 + 无法学层面的转化
- **三项均不满足才触发**
- 以下情况**不触发**：
  1. 跨学科概念已转化为可操作的法学分析步骤（如裁判规则、司法程序步骤）
  2. 概念界定虽跨学科但已有法学层面的操作化步骤（如"移情联想→位阶排序→反思权衡→理由优化"）
  3. 框架虽非纯法学概念，但已提供法学判断标准或裁判适用指引

**framework_unused（score_ceiling=60）**：
- 触发条件：框架核心术语在后文出现少于3次且未被实际调用

## 4. 逻辑严密性维度

### 4.1 核心概念：推理链条

#### 原定义（模糊）
> "推理链条是否不可颠倒"

#### 操作化定义（清晰）

**推理链条严密 = 满足以下标准**：

□ **标准1：前后衔接**
- 检查方法：后一步是否是对前一步的必然回应
- 搜索关键词："因此"、"所以"、"由此可见"、"基于此"
- 正面示例："由于A，因此B；基于B，可以推出C"
- 反面示例："A...B...C..."（无逻辑连接词）

□ **标准2：不可颠倒**
- 检查方法：调换顺序后是否仍然成立
- 测试方法：尝试颠倒相邻两步，看是否合理
- 正面示例：必须先证明A，才能推出B（不可颠倒）
- 反面示例：A和B可以任意顺序论述（可颠倒）

□ **标准3：无跳跃**
- 检查方法：是否存在逻辑跳跃或隐含前提
- 搜索关键词：突然出现的结论，无铺垫的论断
- 正面示例：每个结论都有充分论证
- 反面示例："显然"、"不言而喻"（跳过论证）

□ **标准4：处理反驳**
- 检查方法：是否考虑了反对意见或例外情形
- 搜索关键词："有人可能认为"、"反对意见"、"例外"
- 正面示例："虽然有观点认为XX，但本文认为..."
- 反面示例：完全不提反对意见

**评分标准**：
- 满足 4 项标准 → 80-100 分
- 满足 3 项标准 → 60-79 分
- 满足 2 项标准 → 40-59 分
- 满足 0-1 项标准 → 0-39 分

## 5. 结论可接受性维度

### 5.1 核心概念：制度框架内的可行性

#### 原定义（模糊）
> "对策/结论在制度框架内的可行性"

#### 操作化定义（清晰）

**可行性 = 满足以下至少 2 项标准**：

□ **标准1：法律依据**
- 检查方法：结论是否有明确的法律依据
- 搜索关键词：《XX法》、"第X条"、"法律规定"
- 正面示例："根据《民法典》第XX条，应当..."
- 反面示例："应当..."（无法律依据）

□ **标准2：制度兼容**
- 检查方法：结论是否与现有制度冲突
- 检查内容：是否需要修改多个法律或制度
- 正面示例："该方案可在现有制度框架内实施"
- 反面示例："需要全面修改XX法"（成本过高）

□ **标准3：操作性**
- 检查方法：结论是否具体可操作
- 搜索关键词："具体而言"、"操作步骤"、"实施路径"
- 正面示例："具体操作为：第一步...，第二步..."
- 反面示例："应加强XX"（过于抽象）

□ **标准4：成本合理**
- 检查方法：是否考虑了实施成本和可行性
- 搜索关键词："成本"、"可行性"、"现实条件"
- 正面示例："考虑到现实条件，建议采用XX方案"
- 反面示例：提出理想化方案，不考虑成本

## 6. 前瞻延展性维度

### 6.1 核心概念：延展价值

#### 原定义（模糊）
> "为后续研究'画地图'"

#### 操作化定义（清晰）

**延展价值 = 满足以下至少 1 项标准**：

□ **标准1：指出未来研究方向**
- 检查方法：是否明确指出可以继续研究的问题
- 搜索关键词："未来研究"、"有待进一步"、"可以探讨"
- 正面示例："本文仅讨论了XX，YY问题有待进一步研究"
- 反面示例："本文已全面解决XX问题"（无延展）

□ **标准2：提出新的理论假设**
- 检查方法：是否提出了可供验证的理论假设
- 搜索关键词："假设"、"推测"、"可能"
- 正面示例："基于本文发现，可以推测..."
- 反面示例：只总结已有结论

□ **标准3：方法论贡献**
- 检查方法：是否提出了可供其他研究借鉴的方法
- 搜索关键词："方法"、"路径"、"框架"
- 正面示例："本文提出的XX分析框架可用于YY问题"
- 反面示例：只针对特定问题，无普遍性

□ **标准4：与正文论证的关联**
- 检查方法：延展是否基于正文的论证
- 检查内容：延展是否是正文结论的自然延伸
- 正面示例："基于本文XX结论，可以进一步..."
- 反面示例：延展与正文无关

**评分标准**：
- 满足 3-4 项标准 → 70-100 分
- 满足 2 项标准 → 50-69 分
- 满足 1 项标准 → 30-49 分
- 满足 0 项标准 → 0-29 分

## 7. 使用指南

### 7.1 如何应用这些定义

#### 步骤1：阅读论文
- 快速浏览全文，了解基本结构

#### 步骤2：逐项检查
- 对每个维度，按照操作化定义逐项检查
- 使用搜索关键词定位相关段落
- 记录满足/不满足的标准

#### 步骤3：计算得分
- 根据满足的标准数量，确定分数区间
- 在区间内根据质量微调

#### 步骤4：输出证据
- 引用具体段落作为证据
- 说明为什么满足/不满足某个标准

### 7.2 常见误区

❌ **误区1：凭整体印象评分**
- 错误做法："这篇论文看起来不错，给 75 分"
- 正确做法："满足标准1、2、3，不满足标准4，因此 70 分"

❌ **误区2：过度主观解读**
- 错误做法："我认为这个问题很重要"
- 正确做法："这个问题满足枢纽性标准1和2"

❌ **误区3：忽略证据**
- 错误做法："论文应该有文献综述"
- 正确做法："在第2节找到了文献综述，包含到达点和未竟点"

### 7.3 质量检查

评分完成后，自查以下问题：

□ 每个维度的评分是否基于客观标准？
□ 是否引用了具体段落作为证据？
□ 是否说明了为什么满足/不满足某个标准？
□ 不同维度的评分是否一致（无矛盾）？
