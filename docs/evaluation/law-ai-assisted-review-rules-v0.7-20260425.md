# 法学论文六维 AI 辅助评审规程 v0.7

## 1. 目标

v0.7 的目标是把“专家共识”设为第一约束，把 AI 的角色限定为可验证、可复核、可分流的辅助评价。

- 第一优先：法学评价逻辑可被专家解释和认可。
- 第二优先：AI 执行稳定且可审计。
- 禁止把“模型一致性”当作“法学正确性”替代品。

## 2. 角色定位（不变）

六维名称与边界保持不变，但总分角色明确如下：

- 主评分维度：问题创新性、现状洞察度、分析框架建构力、逻辑严密性。
- 上限维度：结论可接受性（风险筛查 + 总分上限）。
- 弱加分维度：前瞻延展性（条件满足时最多 +5）。

## 3. 总分协议（v0.7 新增硬约束）

### 3.1 核心基础分

```text
基础分 = (问题创新性×30 + 现状洞察度×20 + 分析框架建构力×15 + 逻辑严密性×20) / 85
```

### 3.2 前瞻延展性加分

仅在以下条件全部满足时加分：

- 逻辑严密性 >= 60
- 结论可接受性 >= 60
- 四个主评分维度中不存在 < 50 的塌陷项

加分区间：0 / 2-3 / 4-5（最高 5 分）。

### 3.3 结论可接受性上限

- 结论可接受性 >= 75：不上限
- 60-74：总分上限 75
- < 60：总分上限 65

### 3.4 最终分

```text
最终分 = min(基础分 + 前瞻加分, 结论可接受性上限)
```

## 4. 执行口径修正

- 现状洞察度中，“法学流派数量”只能作辅助信号，不得作为高分充分条件。
- 推荐主口径：可区分的争点结构、解释路径、制度方案分歧、理论立场。
- 逻辑严密性统一使用“充分支撑/可辩护推出/无关键跳步”，避免“必要推出”造成误判。

## 5. AI 输出与专家复核

- AI 必须继续输出六维原始分（0-100）和证据片段，用于横向比较与审计。
- 但最终总分必须按第 3 节协议计算，不能再把六维直接线性加权。
- 遇到结构异常（高创新低逻辑、高结论低论证、高展望低基础等）必须进入人工复核。

## 6. 版本迁移说明

- `law-v2.30-20260425.yaml` 是 v0.7 的首个对齐框架版本。
- 迁移重点是“总分协议对齐”，不是继续扩大 prompt 复杂度。
