### ⚠️ 部分对齐（需后续完善）

#### 2.1 研究创新性的操作化

**现状**：
- v2.19：prompt 中只包含"可争辩性+枢纽性"检查清单（针对问题创新）
- 规则 v0.6：明确提出四项检查清单（创新主张/差异推进/内容观点/方法材料）

**影响**：
- 概念层面已对齐（都理解为"研究创新性"）
- 操作层面未完全对齐（v2.19 的 prompt 未包含内容/观点创新和方法/材料创新的独立检查项）

**建议**：
- 短期：保持现状，v2.19 的"可争辩性+枢纽性"检查清单可以覆盖大部分问题创新的判断
- 长期：考虑将 v2.19 的 prompt 扩展为完整的"研究创新性"三子项检查清单

#### 2.2 逻辑严密性的检查清单

**现状**：
- v2.19：prompt 中只有简化的"链条复述测试+结论推出测试"
- 规则 v0.6：明确提出五项检查清单（前后衔接/推理充分性/概念一致性/材料匹配度/限制条件处理）

**影响**：
- v2.19 的 prompt 未体现"推理充分性"（从"必要性"改为"充分性"）的优化
- v2.19 的 prompt 未明确"方法或材料是否提升论证"归入本维

**建议**：
- 需要更新 v2.19 的逻辑严密性 prompt，补充五项检查清单
- 特别是第 4 项"材料匹配度"需要明确包含"方法或材料是否提升论证"

#### 2.3 结论可接受性的检查清单

**现状**：
- v2.19：未明确检查清单结构
- 规则 v0.6：明确提出三项风险信号筛查

**影响**：
- v2.19 的 prompt 未体现"从 4 项精简为 3 项"的优化
- v2.19 的 prompt 未明确"不重复评价材料匹配度"

**建议**：
- 需要更新 v2.19 的结论可接受性 prompt，明确三项检查清单
- 删除与逻辑严密性重复的"结论与前文推理的一致性"检查项

#### 2.4 前瞻延展性的检查清单

**现状**：
- v2.19：未明确检查清单结构
- 规则 v0.6：明确提出三项路径检查

**影响**：
- v2.19 的 prompt 未体现"从 4 项精简为 3 项"的优化
- v2.19 的 prompt 未明确删除"是否具有继续检验价值"

**建议**：
- 需要更新 v2.19 的前瞻延展性 prompt，明确三项检查清单
- 删除超出 AI 能力边界的"是否具有继续检验价值"检查项

#### 2.5 输出要求的边界说明

**现状**：
- v2.19：所有维度都有 `boundary_notes` 字段
- 规则 v0.6：明确只有 4 个维度需要输出边界说明

**影响**：

**建议**：

- 或者在 v2.19 的 prompt 中明确说明哪些维度需要/不需要输出边界说明
