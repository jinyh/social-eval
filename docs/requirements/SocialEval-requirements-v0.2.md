# SocialEval 系统需求文档

## 背景与目标

当前传统学术评价面临"不可能三角"困境：标准不一（依赖主观经验）、效率低（评审周期漫长）、成本高（专家调度成本巨大）。本项目旨在构建一套以"自主知识体系"为评价底层逻辑的 AI 辅助学术评价系统，以法学论文评审为切入点，最终拓展到人文社科各学科。

**项目名称**：SocialEval — 面向自主知识体系的人工智能辅助学术评价系统

**主要目标**：
- 用 AI 辅助实现高质量、标准统一的学术论文初审
- 以"六维评价框架"量化评分，并提供可解释的评价依据
- 通过多模型并发验证保证评分可靠性
- 提供 Web 界面和 API 接口，支持人工和系统集成两种使用方式

---

## 用户角色与核心诉求

| 角色 | 核心诉求 |
|------|----------|
| **投稿人/研究者** | 上传论文、获取评审进度和最终报告 |
| **期刊编辑委员** | 批量初审、查看完整 AI 评分报告、为待复核论文指定专家、决定是否送外审 |
| **领域专家（评审者）** | 查看 AI 评分报告、进行人工复核、对各维度填写修改意见 |
| **系统管理员** | 配置评价维度/知识体系、管理用户、查看统计数据 |

## 功能需求

### F1. 文档摄取与预处理

- **F1.1** 支持上传 PDF、Word（.docx）、TXT 格式的学术论文
- **F1.2** 预处理分两步：
  - **噪声去除**：自动去除页眉、页脚、致谢等与评价无关内容
  - **参考文献独立提取**：参考文献区块不作为噪声丢弃，而是单独提取存储，供「现状洞察度」评价维度在评分时调用
- **F1.3** 提取论文正文结构（摘要、引言、主体、结论）；
  对于**非标准排版**论文（无明确章节标签），系统自动通过字体规律、编号规律等特征检测章节边界；
  无法可靠识别时降级为"全文分段"模式，并在评审报告中附加结构识别警告标记
- **F1.4** 输出标准化预处理结果，包含：正文核心、参考文献列表（独立字段）、结构识别状态
- **F1.5** 扫描版 PDF 的 OCR 支持为后期功能，待澄清

### F2. 知识体系配置

- **F2.1** 评价框架通过外部 YAML/JSON 配置文件定义，系统动态加载
- **F2.2** 配置内容包括：
  - 学科名称
  - 评价维度列表，每个维度包含：名称、描述、**权重**（0–1 浮点数，所有维度之和须为 1.0）、提示词模板
  - 一致性阈值（标准差上限，可覆盖全局默认值）
- **F2.3** 内置法学评价框架（六维，默认等权），作为默认配置：
  1. **问题创新性**（Problem Originality）
  2. **现状洞察度**（Literature Insight）
  3. **理论建构力**（Theoretical Construction）
  4. **逻辑严密性**（Logical Coherence）
  5. **学术共识度**（Scholarly Consensus）
  6. **前瞻延展性**（Forward Extension）
- **F2.4** 管理员可通过后台界面上传/切换不同学科的知识体系配置文件
- **F2.5** 配置文件支持版本管理（保存历史版本，支持回滚）

### F3. AI 评价引擎

- **F3.1** 支持调用多家商业 AI 模型（OpenAI GPT-4o、Anthropic Claude、DeepSeek 等），通过统一接口抽象切换
- **F3.2** 针对每个评价维度独立构造提示词，AI 逐维度输出**结构化 JSON**，字段包含：
  ```json
  {
    "dimension": "问题创新性",
    "score": 82,
    "evidence_quotes": ["论文原文引用片段1", "..."],
    "analysis": "AI 分析说明文本"
  }
  ```
- **F3.3** 支持"多模型并发评价"模式：同一论文同一维度同时提交给 N 个模型，收集各模型评分
- **F3.4** 支持"智能体研讨会"模式（高级，可配置选项）：
  - 角色设置：主持人、批判者、总结者等多角色
  - **终止条件**（按优先级依次判断）：
    1. 连续两轮评分变化幅度 < 配置阈值（提前收敛，可配置，默认 3 分）
    2. 达到最大轮数上限（可配置，默认 5 轮）后强制结束
  - 强制结束时未收敛的维度标记为「研讨未收敛」，自动进入人工介入队列
- **F3.5** 所有 AI 调用记录（输入/输出/时间戳/模型名称/耗时）须持久化存储，供审计追溯

### F4. 可靠性验证层

- **F4.1** 多模型评分收集完成后，自动计算各维度的均值、标准差
- **F4.2** 一致性阈值判断：
  - 全局默认值：标准差 ≤ 5 分（管理员可在系统设置中修改全局默认值）
  - 各学科可在知识体系 YAML 中单独配置阈值，覆盖全局默认值
  - 达标则标记"高置信度"
- **F4.3** 未达标（分歧较大）的维度/论文自动标记"低置信度"，进入专家复核队列
- **F4.4** 报告中须展示：各模型原始评分、均值 ± 标准差、置信度标签
- **F4.5** 专家复核后，专家评分与 AI 评分**并列存储**，均纳入最终报告展示，不互相覆盖

### F5. 专家复核工作流

- **F5.1** 专家通过系统账号登录后，可查看分配给自己的待复核论文列表
- **F5.2** 专家可查看论文全文（或预处理后的净化版本）及 AI 评分报告（含各模型原始评分）
- **F5.3** 专家可对每个维度修改评分，并填写修改理由（文本，必填）
- **F5.4** **专家分配机制**：编辑在论文详情页手动指定一名或多名平台专家账号；指定完成后，系统自动向被指定专家发送邮件通知（含论文摘要和 AI 评分摘要）
- **F5.5** 专家可通过邮件中的链接直接跳转到复核页面（需已登录或跳转后提示登录）
- **F5.6** 复核结果需记录：专家身份、操作时间、各维度修改前/后评分、修改理由

### F6. 评审报告生成

- **F6.1** 每次评审任务完成后生成标准报告，包含以下字段：

  | 字段 | 说明 |
  |------|------|
  | 六维 AI 评分 | 各维度：均值 ± 标准差，置信度标签 |
  | 综合加权总分 | 各维度均值按 YAML 配置权重加权求和（0-100 分） |
  | AI 评价依据 | 每维度：论文原文引用片段 + AI 分析说明（来自 F3.2 JSON） |
  | 多模型一致性数据 | 各模型原始评分列表、均值、标准差 |
  | 六维雷达图 | 以各维度均值绘制（可视化） |
  | 专家复核意见 | 如有：专家评分（并列展示，不覆盖 AI 评分）、修改理由、复核时间 |

- **F6.2** 支持导出 PDF 格式报告（供人工阅读）
- **F6.3** 支持导出 JSON 格式原始数据（供第三方系统集成）

### F7. Web 用户界面

- **F7.1** **投稿人界面**：论文上传、进度查看、最终报告下载
  - 投稿人**仅可查看最终报告**（含综合加权总分、六维雷达图、文字评语），不展示 AI 各模型原始评分及多模型一致性数据
- **F7.2** **编辑界面**：论文列表管理、批量初审视图、单篇完整报告详情、专家指定操作
- **F7.3** **专家界面**：复核任务列表、含 AI 完整评分的评分修改页面
- **F7.4** **管理员界面**：用户管理、知识体系配置上传、一致性阈值全局设置、系统统计看板
- **F7.5** 所有界面需要身份认证（登录/权限控制）

### F8. API 接口

- **F8.1** 提供 RESTful API，支持第三方系统集成（如期刊投稿管理系统）
- **F8.2** 核心 API 端点：
  - `POST /api/papers` — 上传论文，返回任务 ID
  - `GET /api/papers/{id}/status` — 查询评审进度
  - `GET /api/papers/{id}/report` — 获取评审报告（JSON）
  - `POST /api/papers/batch` — 批量提交论文任务
- **F8.3** API 需要 Token 认证（API Key 机制）
- **F8.4** 批量任务异步处理，通过回调 URL 或轮询接口通知完成状态

## 非功能需求

### 性能
- 单篇论文（2万字以内）AI 评审完成时间：≤ 10 分钟
- API 批量任务支持并发处理，单次批量上限 ≥ 100 篇
- Web 界面核心页面加载时间：≤ 3 秒

### 可靠性
- 系统可用性：≥ 99%（工作时间）
- AI 调用失败时自动重试（最多 3 次），失败后标记任务异常并通知管理员

### 安全性
- 所有接口 HTTPS 加密传输
- 用户身份认证（JWT 或 Session）
- API Key 权限隔离
- 论文数据允许云端存储及调用外部 AI API（数据不做本地隔离要求）
- 投稿人不可访问 AI 原始调用数据和多模型分歧数据

### 可扩展性
- 知识体系通过外部配置文件管理，新增学科无需改代码
- AI 模型接口通过统一抽象层管理，新增模型只需扩展配置
- 后期可将各模块拆分为独立微服务
